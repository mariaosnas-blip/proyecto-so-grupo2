Conclusiones

01

Integración de tecnologías modernas

A lo largo de este proyecto logramos construir una infraestructura completa que integra virtualización, contenedores y orquestación. Implementamos dos VMs Linux con comunicación SSH, desplegamos servicios con Docker Compose usando Rocky Linux como imagen base, y orquestamos réplicas de Nginx con Kubernetes Minikube. Esta experiencia nos permitió comprender cómo estas tecnologías se complementan en entornos reales de producción.

02

Dificultades con SSH y red en Docker

Uno de los desafíos más significativos fue la configuración del acceso SSH entre las VMs. Al ejecutar ssh usuario@192.168.100.20 recibimos el error Connection refused, indicando que el servicio no estaba activo en Rocky Linux. Fue necesario instalar openssh-server, iniciarlo con systemctl y abrir el puerto en el firewall con firewall-cmd. Aprendimos que una red funcional no garantiza que los servicios estén disponibles; cada uno debe configurarse explícitamente.

En Docker, enfrentamos dificultades para que el frontend y el backend se comunicaran dentro de la red de contenedores. Comprendimos que Docker maneja su propia capa de red interna y que es necesario definir correctamente los puertos y dependencias en docker-compose.yml para garantizar comunicación estable entre servicios.

03

Documentación y trabajo colaborativo

Aprendimos que dividir responsabilidades y documentar cada paso es tan importante como el trabajo técnico mismo. Publicar el sitio en GitHub Pages y mantener un repositorio con commits de todos los integrantes nos enseñó buenas prácticas de colaboración que aplicaremos en proyectos profesionales futuros.
